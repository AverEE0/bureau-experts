# Как выложить проект на GitHub и открыть приложение по ссылке

Пошаговая инструкция: создание репозитория, загрузка кода и деплой, чтобы приложение можно было открыть в браузере по ссылке.

---

## Часть 1. Репозиторий на GitHub

### Шаг 1. Аккаунт GitHub

- Если аккаунта нет: [github.com](https://github.com) → **Sign up**.
- Если есть: войдите в аккаунт.

### Шаг 2. Создать новый репозиторий

1. В правом верхнем углу нажмите **+** → **New repository**.
2. **Repository name:** например, `bureau-experts` (латиница, без пробелов).
3. **Description:** по желанию, например: «Экспертная платформа СЭЦ БЮРО ЭКСПЕРТОВ».
4. Выберите **Public**.
5. **Не ставьте** галочки «Add a README», «Add .gitignore», «Choose a license» — в проекте они уже есть.
6. Нажмите **Create repository**.

На следующей странице будет URL репозитория, например:  
`https://github.com/ваш_логин/bureau-experts`

---

## Часть 2. Загрузить код с компьютера

### Шаг 1. Открыть терминал в папке проекта

- **Windows:** в проводнике откройте папку `bureau-app` (та, где лежат `package.json` и `src`), в адресной строке введите `cmd` или `powershell` и нажмите Enter.
- Либо в Cursor/VS Code откройте папку `bureau-app` и откройте встроенный терминал (Terminal → New Terminal).

### Шаг 2. Проверить, что Git установлен

Выполните:

```bash
git --version
```

Если команда не найдена — установите Git: [git-scm.com](https://git-scm.com) (Windows) и снова откройте терминал.

### Шаг 3. Настроить Git (один раз на компьютере)

Укажите имя и email для коммитов (email должен совпадать с аккаунтом GitHub или быть добавлен в настройках GitHub):

```bash
git config --global user.email "omashi001@gmail.com"
git config --global user.name "Ваше Имя"
```

**Важно:** при `git push` GitHub **не принимает обычный пароль**. Нужен **Personal Access Token (токен)**:
1. GitHub → правый верхний угол → **Settings** → внизу слева **Developer settings** → **Personal access tokens** → **Tokens (classic)**.
2. **Generate new token (classic)** → отметьте право **repo** → сгенерируйте и **скопируйте токен** (он показывается один раз).
3. При первой команде `git push` введите логин GitHub и **вместо пароля вставьте этот токен**. Сохраните токен в надёжном месте (менеджер паролей); в чатах и в коде его хранить нельзя.

### Шаг 4. Инициализация и первый коммит

Выполните по очереди. Если репозиторий создаётся из папки **6256-main** (родительская), перейдите в неё; если только из **bureau-app** — откройте её:

```bash
cd путь\к\6256-main
REM или: cd путь\к\6256-main\bureau-app
git init
git add .
git status
git commit -m "Initial commit: SEC BJURO EKSPERTOV - ekspertnaya platforma"
git branch -M main
```

Если `git status` показывает только нужные файлы (нет лишних паролей и т.п.) — можно делать `commit`.

**Если коммит пишет «Author identity unknown»** — сначала выполните Шаг 3 (git config user.email и user.name).

### Шаг 5. Привязать репозиторий и отправить код

Замените `ВАШ_ЛОГИН` и `ИМЯ_РЕПОЗИТОРИЯ` на свои (как в URL репозитория):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
git push -u origin main
```

- Если попросят логин/пароль: введите данные GitHub. Для пароля может потребоваться [Personal Access Token](https://github.com/settings/tokens) вместо обычного пароля.
- После успешного `push` код будет на GitHub. Открыть репозиторий можно по ссылке:  
  `https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ`

## Часть 3. Открыть приложение по ссылке (деплой на Vercel)

Так вы получите ссылку вида `https://ваш-проект.vercel.app`, по которой приложение открывается в браузере.

### Шаг 1. Регистрация на Vercel

1. Перейдите на [vercel.com](https://vercel.com).
2. Нажмите **Sign Up** и выберите **Continue with GitHub**.
3. Разрешите Vercel доступ к вашему аккаунту GitHub.

### Шаг 2. Импорт проекта

1. Нажмите **Add New…** → **Project**.
2. В списке репозиториев выберите **ИМЯ_РЕПОЗИТОРИЯ** (тот, куда вы залили `bureau-app`).
3. **Root Directory:** если при создании репозитория вы загружали всю папку **6256-main**, укажите `6256-main/bureau-app`. Если загружали только содержимое папки **bureau-app** (в корне репозитория лежат `package.json` и `src`), оставьте поле пустым.
4. **Framework Preset:** Vercel обычно подставляет **Create React App** — оставьте как есть.
5. **Build Command:** `npm run build`  
   **Output Directory:** `build`  
   (часто уже подставлены по умолчанию).
6. Нажмите **Deploy**.

### Шаг 3. Открыть по ссылке

Через 1–2 минуты появится страница с ссылкой вида:

`https://bureau-experts-xxxx.vercel.app`

Откройте её в браузере — откроется ваше приложение. Эту ссылку можно сохранить или отправить кому-то для просмотра.

При следующих обновлениях кода достаточно сделать `git push` в репозиторий — Vercel по желанию можно настроить на автоматический деплой при каждом push (обычно это уже включено).

---

## Часть 4. Альтернатива: Netlify

1. Зайдите на [netlify.com](https://netlify.com), зарегистрируйтесь (можно через GitHub).
2. **Add new site** → **Import an existing project** → **Deploy with GitHub** → выберите репозиторий.
3. Укажите:
   - **Build command:** `npm run build`
   - **Publish directory:** `build`
4. **Deploy site**. Ссылка вида `https://название.netlify.app` откроет приложение.

## Итог

| Действие              | Результат |
|-----------------------|-----------|
| Выполнить Часть 1–2   | Код на GitHub, можно смотреть и клонировать по ссылке на репозиторий. |
| Выполнить Часть 3 (или 4) | Приложение открывается в браузере по постоянной ссылке (Vercel или Netlify). |
