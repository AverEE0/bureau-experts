# Требования к платформе «БЮРО ЭКСПЕРТОВ»

Краткая сводка по **ТЗ** и **переписке Telegram** (ChatExport): что учтено в интерфейсе и что планируется.

---

## 1. Из технического задания

- **CRM**: регистрация физ/юр лиц, проверка ИНН/ОГРН (ФНС), единая карточка, роли, каналы связи по телефону.  
  *В интерфейсе: раздел «Клиенты» с таблицей, поиск, фильтры, добавление клиента.*

- **Документооборот и OCR**: загрузка сканов/PDF, классификация (Договор, Счёт, Акт, Отчёт, Заключение), извлечение реквизитов.  
  *В интерфейсе: «Документы» — загрузка, выбор типа, журнал, этапы загрузка → классификация → распознавание.*

- **Бизнес-циклы**:  
  - Оценка: Запрос → Договор → Счёт → Отчёт/Выписка/Акт/Справка → Акт выполненных работ → Архив.  
  - Экспертиза: Запрос → Договор → Счёт → Заключение → Акт → Архив.  
  *В интерфейсе: разделы «Оценка» и «Экспертиза» с описанием циклов и типов документов.*

- **Шаблоны**: отчёты (квартира, дом, ТС), договоры, акты. Поля для подстановки, даты.  
  *В интерфейсе: «Шаблоны» — список шаблонов из переписки и описание полей.*

- **Архив и реестр**: сроки хранения, поиск по ФИО/адресу/кадастру/номеру дела; настраиваемые столбцы реестра.  
  *В интерфейсе: «Архив» → «Реестр» — таблица с включением/выключением столбцов и поиском.*

- **Omnichannel**: Telegram, MAX, BIP, SMS, Email, звонки; логи в карточке клиента.  
  *В интерфейсе: пункты меню по каналам; в Telegram — модальное окно «Отправить сообщение» (получатель + текст).*

- **Безопасность и стек**: ФЗ-152, ФЗ-73, ФЗ-135; React + Ant Design (frontend), FastAPI + PostgreSQL (backend по ТЗ).  
  *В интерфейсе: брендинг и разделы под эти требования; backend и шифрование — отдельный этап.*

---

## 2. Из переписки (ChatExport)

- **Интеграция Telegram**: сообщения из Telegram к сотруднику в систему, интеграция систем.  
  *Учтено: раздел Telegram с модальным окном отправки сообщения (получатель, текст).*

- **Документы**: Договор БЭ (оценка), Акт приема-передачи, отчёт (земельный участок и др.), Счет № 101, Акт выполненных работ. Поля в шаблонах выделены (красным/жёлтым) — извлекать или вводить вручную; даты — автоматом текущая с возможностью замены (в т.ч. задним числом).  
  *Учтено: типы в «Документах» и «Шаблонах», описание полей и дат.*

- **Фото в отчёт/выписку/заключение**: 2–3 фото в строку на лист А4, компактно на весь лист.  
  *Учтено: в «Оценка» и «Экспертиза» указано про вставку фото.*

- **OCR vs локальная ИИ**: в переписке обсуждалась локальная модель для извлечения/подстановки данных и работы с CRM; данные не уходят в интернет.  
  *Учтено как целевая архитектура; в UI — этапы документооборота и связь с CRM.*

- **Реестр**: аналог «Лист XLS» / СЭЦ — таблица с возможностью включать и отключать столбцы.  
  *Учтено: страница «Реестр» с настраиваемыми столбцами.*

- **Шаблоны из переписки**: шаблон_квартира, шаблон_дом, шаблон_ТС_AL, ТС_краткий, Договор_БЭ_№ оценка, АКТ приема-передачи.  
  *Учтено: список в «Шаблоны» с именами файлов и назначением.*

- **Законодательство и СРО**: подстановка в заключения, периодическая проверка изменений.  
  *Учтено: в «Экспертиза» указаны ФЗ-73, ФЗ-135, СРО.*

## 3. Что сделано в интерфейсе

| Раздел        | Реализовано                                                                 |
|---------------|-------------------------------------------------------------------------------|
| CRM / Клиенты | Таблица, поиск, фильтр по статусу, добавление клиента, статистика            |
| Документы     | Загрузка файла, тип по умолчанию, журнал, кнопки «Классифицировать»/«Распознать» |
| Шаблоны       | Список шаблонов (квартира, дом, ТС, договор, акт, счёт/акт), поля подстановки |
| Оценка        | Цикл, документы (отчёт, выписка, акт, справка), фото 2–3 в строку, методы   |
| Экспертиза    | Цикл, заключение с фото, законодательство, виды экспертиз                   |
| Архив → Реестр| Таблица с переключаемыми столбцами, поиск по реестру                         |

## 4. По фотографиям из переписки (изучено)

- **Скрин интерфейса Telegram**: модальное окно «Отправить сообщение в Telegram» — поля «Получатель» (выбор из списка) и «Сообщение» (текст). Учтено в разделе Telegram.
- **Договор оказания услуг**: образец ДОГОВОР ОКАЗАНИЯ УСЛУГ №2511ОД/11.2025, ООО «БЮРО ЭКСПЕРТОВ», ген. директор Евтеев Д.А., г. Мариуполь, предмет договора. Учтено в шаблонах и типах документов.
- **Паспорт (место жительства)**: поля для извлечения выделены — регион, пункт, улица, дом, квартира; дата регистрации, МВД. Учтено: в документообороте нужны структурированные поля адреса и реквизитов для подстановки в отчёты/договоры.
- **Выписка ЕГРН**: кадастровый номер, адрес объекта, площадь (м²), назначение (жилое и т.д.), год постройки, статус записи. Поля, подчёркнутые в образце, — ключевые для отчётов об оценке и заполнения из ЕГРН/Росреестра. Учтено в разделе «Оценка» и в реестре (кадастровый номер, адрес).
- **Полис-оферта (страхование ответственности оценщика)**: пример документа с реквизитами страховщика, банка, страхователя; даты, суммы. Подтверждает потребность в извлечении реквизитов и дат из разных типов документов.

Дальнейшие шаги по ТЗ: backend (FastAPI, PostgreSQL), реальный OCR/ИИ, интеграции (ГАС, Росреестр, ОФД, ЭДО, 1С, банки, мессенджеры), 2FA, шифрование, деплой.
