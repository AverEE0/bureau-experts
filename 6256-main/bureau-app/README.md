# СЭЦ «БЮРО ЭКСПЕРТОВ» — Экспертная платформа

Веб-приложение для судебно-экспертного центра: CRM, документооборот, судебная экспертиза, оценка, интеграции с ГАС «Правосудие», Мой Арбитр и др.

**Стек:** React 19, Ant Design 6, Create React App.

---

## Быстрый старт

```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки (по умолчанию http://localhost:3000)
npm start
```

Если порт 3000 занят, создайте файл `.env` в корне проекта и укажите:
```
PORT=3001
```
Тогда приложение откроется на http://localhost:3001

На Windows можно запустить `ЗАПУСК.bat` — он стартует приложение с портом 3001.

---

## Сборка для продакшена

```bash
npm run build
```

Папка `build/` будет содержать статику для размещения на любом хостинге (Vercel, Netlify, nginx и т.д.).

## Структура приложения

- **Дашборд** — сводка и быстрый доступ.
- **CRM** — клиенты, сделки, контакты, история.
- **Документооборот** — документы, OCR, шаблоны, подписи.
- **Бизнес-циклы** — судебная экспертиза (в т.ч. судебное дело, вид экспертизы, номер дела, синхронизация с ГАС «Правосудие» / Мой Арбитр), оценка, юридические услуги, недвижимость.
- **Архив** — документы, медиа, история, реестр с настраиваемыми столбцами.
- **Финансы** — ОФД, ЭДО, 1С, банки.
- **Omnichannel** — Telegram, MAX, BIP, SMS, Email, звонки.
- **Отчёты** — финансовые, операционные, аналитика.
- **Настройки** — данные организации, безопасность, уведомления, интеграции, документы, внешний вид.
- **Внутренняя переписка** — чат.

Текущая версия — **фронтенд-прототип** (без бэкенда и реальных API). Полное ТЗ и план интеграций см. в `REQUIREMENTS.md` и `ТЗ-соответствие.md`.

## Как выложить на GitHub и открыть по ссылке

### 1. Создать репозиторий на GitHub

1. Зайдите на [github.com](https://github.com), войдите в аккаунт.
2. Нажмите **New repository** (или **+** → New repository).
3. Укажите имя (например, `bureau-experts`), описание по желанию, **не** включайте README (он уже есть в проекте).
4. Нажмите **Create repository**.

### 2. Загрузить проект с компьютера

В терминале перейдите в папку проекта и выполните:

```bash
cd путь\к\6256-main\bureau-app

git init
git add .
git commit -m "Initial commit: СЭЦ БЮРО ЭКСПЕРТОВ — экспертная платформа"

git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
git push -u origin main
```

Подставьте вместо `ВАШ_ЛОГИН` и `ИМЯ_РЕПОЗИТОРИЯ` свои данные (например, `ivanov` и `bureau-experts`). Если GitHub попросит авторизацию — введите логин и пароль (или токен).

После этого код будет на GitHub, и по ссылке вида  
`https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ`  
можно открыть репозиторий и просматривать проект.

### 3. Открыть приложение по ссылке в браузере (деплой)

Чтобы не только смотреть код, но и **открыть само приложение по ссылке** (как сайт), удобно использовать бесплатный хостинг:

#### Вариант A: Vercel (рекомендуется)

1. Зарегистрируйтесь на [vercel.com](https://vercel.com) (можно через аккаунт GitHub).
2. Нажмите **Add New** → **Project** и импортируйте ваш репозиторий с GitHub.
3. Vercel сам определит Create React App. Оставьте **Root Directory** пустым, если репозиторий содержит только папку `bureau-app`; если в репозитории корень = папка приложения, тоже оставьте как есть.
4. Нажмите **Deploy**. Через 1–2 минуты появится ссылка вида `https://ваш-проект.vercel.app` — по ней можно открыть приложение в браузере.

#### Вариант B: Netlify

1. Зарегистрируйтесь на [netlify.com](https://netlify.com), подключите GitHub.
2. **Add new site** → **Import an existing project** → выберите репозиторий.
3. Build command: `npm run build`, Publish directory: `build`.
4. **Deploy**. Ссылка вида `https://ваш-проект.netlify.app` откроет приложение.

Подробная пошаговая инструкция с скриншотами — в файле **GITHUB_И_ДЕПЛОЙ.md**.
