# Соответствие техническому заданию

Проверка по разделам ТЗ: что реализовано в интерфейсе (frontend), что только задекларировано, что отсутствует.

---

## 1. Общие положения (п. 1 ТЗ)

| Требование | Статус |
|------------|--------|
| Наименование проекта «БЮРО ЭКСПЕРТОВ — Единая цифровая платформа…» | ✅ Логотип и брендинг в интерфейсе |
| Цель: автоматизация, безопасность, минимальные затраты, работа в РФ, гибкость | ✅ Отражено в подписи футера (ФЗ-152, ФЗ-73, ФЗ-135) и структуре разделов |

---

## 2. Функциональные требования

### 2.1. Управление клиентской базой (CRM)

| Требование ТЗ | Статус |
|---------------|--------|
| Регистрация клиентов (физ/юр) с проверкой по ИНН/ОГРН через ФНС | ⚠️ Частично: форма добавления клиента (тип, ИНН, контакты), **без реального запроса к ФНС** |
| Единая карточка клиента с историей обращений, договоров, платежей | ❌ Нет: есть только таблица клиентов, **нет отдельной карточки клиента** с историей |
| Ролевая модель: администратор, менеджер, оценщик, эксперт, клиент | ❌ Нет в интерфейсе |
| Автоопределение каналов связи по номеру телефона | ❌ Нет |

### 2.2. Документооборот и распознавание

| Требование ТЗ | Статус |
|---------------|--------|
| Загрузка документов (сканы, фото, PDF) через веб, email, мессенджеры | ⚠️ Частично: **только веб-загрузка** в разделе «Документы», без email/мессенджеров |
| OCR на базе Tesseract с дообучением | ❌ Нет: только кнопки «Классифицировать» / «Распознать» с имитацией, **нет бэкенда OCR** |
| Классификация: Договор, Счет, Акт, Отчет, Заключение эксперта, Заключение судебного эксперта | ✅ Типы в выборе при загрузке и в журнале документов |
| Извлечение структурированных данных (реквизиты, адреса, суммы, объекты оценки) | ⚠️ Описано в блоке «Извлекаемые поля» (паспорт, ЕГРН, договор), **реального извлечения нет** |

### 2.3. Автоматизированные бизнес-циклы

| Цикл / Требование ТЗ | Статус |
|----------------------|--------|
| **Цикл 1. Судебная экспертиза**: Запрос → Договор → Счёт → Экспертиза → Акт → Архив | ✅ Описан на странице «Экспертиза» |
| Интеграция с ГАС «Правосудие» | ❌ Нет |
| Генерация заключения с законодательством, фото 2–3 в строку на А4 | ✅ Указано в тексте раздела; **генерации документов нет** |
| **Цикл 2. Оценка недвижимости**: Запрос → Договор → Счёт → Отчёт/Выписка/Акт/Справка → Акт → Архив | ✅ Описан на странице «Оценка» |
| Данные из CRM, ЕГРН/Росреестра, аналоги, законодательство, фото в отчёт | ✅ Описано; **интеграций и генерации нет** |
| **Цикл 3. Юридическое сопровождение**: шаблоны ГПК/АПК/КоАП, календарь сроков | ⚠️ Есть страница «Юридические услуги», **без шаблонов и календаря** |
| **Цикл 4. Риэлторская оценка**: парсинг Авито, ЦИАН, Яндекс.Недвижимость, VK; интеграция с бюроэкспертов.рф | ⚠️ Есть страница «Недвижимость», **без парсинга и интеграций** |

### 2.4. Omnichannel-связь с клиентами

| Требование ТЗ | Статус |
|---------------|--------|
| Каналы: Telegram, MAX, BIP, SMS, Email, голосовые звонки (WebRTC) | ✅ Отдельные пункты меню и страницы для каждого канала |
| Отправка сообщения в Telegram (получатель + текст) | ✅ Модальное окно на странице Telegram |
| Автоопределение каналов по телефону | ❌ Нет |
| Логирование коммуникаций в карточке клиента | ❌ Нет (нет и самой карточки клиента) |
| Шаблоны сообщений по этапам циклов | ❌ Нет |

### 2.5. Генерация экспертных заключений

| Требование ТЗ | Статус |
|---------------|--------|
| Подстановка актуального законодательства (ФЗ-73, ФЗ-135, методички) | ✅ Упомянуто на странице «Экспертиза» |
| Типы: товароведческая, строительно-техническая, землеустроительная, судебно-бухгалтерская | ✅ Перечислены на странице |
| Версионность законодательства на дату заключения | ❌ Нет в интерфейсе |
| Электронная подпись (КЭП/УКЭП) через КриптоПро | ❌ Нет; есть только раздел «Подписи» в меню |

### 2.6. Архивная система

| Требование ТЗ | Статус |
|---------------|--------|
| Зашифрованное хранилище документов | ❌ Нет (только frontend) |
| Сроки хранения: 75 / 10 / 5–10 / 3 года по типам | ❌ Нет в интерфейсе |
| Соответствие ФЗ-125 и Приказа Минкультуры №558 | ❌ Нет |
| Поиск по ФИО, адрес, кадастровый номер, номер дела | ✅ Реестр: поиск по реестру + столбцы (ФИО/заказчик, адрес, кадастр, № дела) |
| Экспорт в PDF/A, XML для госархивов | ❌ Нет |

### 2.7. Финансовая интеграция

| Требование ТЗ | Статус |
|---------------|--------|
| ОФД (онлайн-кассы, фискализация, чеки 54-ФЗ) | ⚠️ Страница «ОФД» есть, **интеграции нет** |
| ЭДО (СБИС, Контур.Диадок, 1С-ЭДО, КЭП/УКЭП) | ⚠️ Страница «ЭДО» есть, **интеграции нет** |
| 1С:Предприятие (контрагенты, договоры, счета, акты, зарплата) | ⚠️ Страница «1C» есть, **интеграции нет** |
| Банки (ЮKassa, Cloud Payments, СБП, платежные поручения, Клиент-Банк) | ⚠️ Страница «Банки» есть, **интеграции нет** |

## 3. Нефункциональные требования

| Требование ТЗ | Статус |
|---------------|--------|
| Безопасность: Zero Trust, TLS 1.3, AES-256, LUKS, 2FA, WAF, Fail2ban | ❌ Нет (нет backend и инфраструктуры) |

| Резервное копирование (ежедневно + облако) | ❌ Нет |
| Адаптивный дизайн (ПК, планшет, смартфон) | ✅ Адаптивная вёрстка, мобильное меню (Drawer) |
| Браузеры: Яндекс, Chrome, Firefox, Edge, Safari | ✅ Обычный React-проект, совместим с перечисленными |
| Локализация: русский язык, форматы дат/валют РФ | ✅ Интерфейс на русском |

|-----------------|--------|

## 5. Итоговая сводка

| Категория | Реализовано в UI | Частично / только описание | Не реализовано |
|-----------|------------------|----------------------------|----------------|
| **Структура и навигация** | Все разделы меню по ТЗ, Дашборд, Реестр | — | — |
| **CRM** | Таблица клиентов, поиск, фильтр, добавление | Форма без проверки ФНС | Карточка клиента, роли, каналы по телефону |
| **Документооборот** | Загрузка, типы документов, журнал, извлекаемые поля | — | OCR, email/мессенджеры, реальное извлечение |
| **Бизнес-циклы** | Описание циклов и документов (Оценка, Экспертиза) | Страницы Юр. услуги, Недвижимость | ГАС, Росреестр, парсинг площадок, генерация документов |
| **Шаблоны** | Список шаблонов (квартира, дом, ТС, договор, акт) | — | Редактор, подстановка из CRM/ЕГРН |
| **Omnichannel** | Страницы каналов, модалка «Отправить в Telegram» | — | API каналов, логи в карточке, шаблоны сообщений |
| **Архив** | Реестр с поиском и настраиваемыми столбцами | Страницы Документы, Медиа, История | Шифрование, сроки хранения, экспорт PDF/A, XML |
| **Финансы** | Страницы ОФД, ЭДО, 1С, Банки | — | Все интеграции |
| **Отчёты** | Страницы Финансовые, Операционные, Аналитика | — | Реальные отчёты и данные |
| **Backend и инфраструктура** | — | — | FastAPI, PostgreSQL, OCR, очереди, безопасность, деплой |

**Вывод:** реализован **фронтенд-прототип** по ТЗ: все разделы из ТЗ присутствуют в меню и имеют страницы, часть страниц содержит осмысленный контент (CRM-таблица, документооборот с журналом, шаблоны, оценка, экспертиза, реестр). **Не реализованы:** backend, интеграции (ФНС, ГАС, Росреестр, ОФД, ЭДО, 1С, банки, мессенджеры), реальные OCR/генерация документов, карточка клиента с историей, роли, шифрование архива, 2FA и остальная инфраструктура из п. 3–4 ТЗ.
